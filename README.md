# Delta-Neutral ETH Carry Strategy

An on-chain proof-of-concept and real-data backtest of a delta-neutral long-spot / short-perpetual ETH carry strategy.

**Research question:** Does a delta-neutral long-spot / short-perpetual ETH strategy generate attractive historical returns through funding carry, and how can the mechanism be represented on-chain?

---

## What This Project Does

The strategy holds two legs simultaneously:

| Leg | Position | Direction |
|-----|----------|-----------|
| Spot | ETH/USDT spot exposure (backed by USDC) | Long |
| Perp | ETHUSDT perpetual futures | Short |

Because the legs are equal in notional and opposite in direction, **ETH price moves cancel out** (delta-neutral). The only remaining income is the perpetual funding rate — the periodic payment that long traders pay to shorts when the market is in contango (perp price > spot price).

**Carry = Funding income − Benchmark opportunity cost − Execution costs**

The project is split into two independent layers:

1. **Backtest layer** — runs on real historical data (Coinbase spot + Deribit perp, via CoinAPI) to show what this strategy would have earned historically
2. **Smart-contract layer** — an on-chain proof-of-concept that demonstrates the mechanism correctly in Solidity

## Quick Start

### Prerequisites

```bash
npm install
pip install pandas numpy matplotlib
```

Copy `.env.example` to `.env` and fill in your keys (for Sepolia deployment).

### 1 — Run the smart-contract tests

```bash
npx hardhat test
```

Expected output: **76 tests passing**. Demonstrates the full on-chain lifecycle including delta-neutral verification.

### 2 — Run the backtest (requires real data)

Place a combined hourly CSV (Coinbase spot + Deribit perp) in `data/raw/`.
See [data/README.md](data/README.md) for the expected file format. Then:

```bash
python3 backtest/run_backtest.py
```

Outputs:
- `output/tables/backtest_hourly.csv` — hourly P&L breakdown
- `output/tables/backtest_metrics.csv` — summary metrics
- `output/charts/chart1_cumulative_pnl.png` — cumulative net P&L
- `output/charts/chart2_drawdown.png` — drawdown chart
- `output/charts/chart3_pnl_decomposition.png` — spot / perp / funding decomposition

To run the full methodology audit:

```bash
python3 backtest/audit.py
```

### 3 — Deploy to local node

```bash
npx hardhat node                                              # Terminal 1
npx hardhat run scripts/deploy-local.ts --network localhost  # Terminal 2
```

### 4 — Deploy to Sepolia

```bash
npx hardhat run scripts/deploy-sepolia.ts --network sepolia
npx hardhat run scripts/verify-sepolia.ts  --network sepolia
```

## Strategy Details

### Carry score formula

```
carryScore = (dailyFundingRate × 365) − benchmarkRate − costs
```

Entry is allowed only when `carryScore > leverageScaledThreshold`:

```
threshold = costs + (leverageRatio × riskPremium)
```

Example at 3 bps/day funding, 2% benchmark, 0.5% costs, 5x leverage:
```
carryScore = 1095 − 200 − 50 = 845 bps
threshold  = 50 + 5 × 75     = 425 bps
→ Entry permitted (845 > 425)
```

### Auto-exit conditions

The vault exits automatically if any of these fire:

1. **MARGIN** — perp margin ratio drops below 800 bps (liquidation protection)
2. **CAPITAL** — net loss exceeds 10% of posted collateral
3. **CARRY** — funding no longer covers benchmark + costs
4. **TIME** — position held longer than 30 days

### Delta-neutrality on-chain

`StrategyVault.getVaultState()` computes both legs and reports `netDeltaPnL = spotPnL + shortPerpPnL`. At equal notional, this is approximately zero — verified in `test/integration/FullLifecycle.test.ts`.

## What Is Mock vs Real

| Component | Real or Mock |
|-----------|-------------|
| Backtest spot price | **Real** — Coinbase ETH/USD, 1-hour bars (CoinAPI) |
| Backtest perp price | **Real** — Deribit ETH/USD perpetual, 1-hour bars (CoinAPI) |
| Funding proxy | **Derived** — lagged `basis_pct / 100 / 8` per hourly bar |
| ETH/USD oracle (Sepolia) | **Real** — Chainlink ETH/USD feed |
| Perpetual exchange | **Mock** — `MockPerpEngine` (no real order book) |
| USDC | **Mock** — `MockUSDC` (mintable for testing) |
| Lending yield | **None** — not part of this strategy |

---

## Limitations

- `MockPerpEngine` does not model slippage, counterparty risk, or liquidation mechanics of a real exchange
- Backtest assumes 1:1 hedge ratio with no rebalancing
- No live market data feed — backtest runs from local CSV files
- The on-chain vault does not connect to the backtest; they are independent layers
